# MD.Piece UI & 配色研究（v1 · 2026-05）

> 本文件是 MD.Piece UI 改版的設計指引，是 CSS token 與 elder-mode、severity color
> 等實作的單一事實來源。每次重大 UI 變動前請對照本文件，必要時更新版本號。

## 1. 主色與輔色（已落地，不再變更）

| Token        | Hex       | 含意                                                            |
|--------------|-----------|---------------------------------------------------------------|
| `--accent`   | `#4A90C2` | **醫療藍**：CalmBlue/Apple Health 共識色；冷靜、可信、不刺激。           |
| `--teal`     | `#5BB5A8` | **Sage Teal**：療癒輔色；用於「進度」「自我照護」「正向行為」回饋。              |
| `--success`  | `#6FBF8B` | 薄荷綠：正向結果、達標、穩定。                                            |
| `--warning`  | `#E8B568` | 暖琥珀：「需注意」但非急；不會引發焦慮的提示色。                                |
| `--danger`   | `#D85F5F` | 清晰紅：急診、藥物衝突；**只能用「點」不能用「面」**，避免「過年感」與焦慮升級。         |
| `--klein`    | `#1F3D58` | 深海軍藍：強調文字、邊界；維持高對比。                                       |
| `--text`     | `#1F3D58` | 內文：在 `--bg-deep`（#FFFFFF）上對比度 12.6 : 1，遠優於 WCAG AAA。 |

### 設計憲法檢核

- 第 2 條「可信任、可解釋的 AI」：藍 + 綠的醫療色系符合患者調查中「最不引起焦慮」的配色（KLAS 2025、PwC Health Consumer Survey 2025）。
- 第 7 條「本地化與文化敏感」：避開純黑底（在台灣文化中與喪儀關聯）；避開大紅大金（過年感）。

## 2. 字級與行高（繁中比英文站 +1 階）

繁體中文筆畫密集，相同字級時實際可讀面積比英文小。研究建議在 mobile-first 設計中
**比 Material/HIG 預設大 1 階字級**，並提供長者模式再 +1 階。

| Token        | 預設 (px) | 長者模式 (px) | 用途                |
|--------------|----------|---------------|---------------------|
| `--font-xs`  | 13       | 14            | 標籤、metadata       |
| `--font-sm`  | 14       | 16            | 次要內文             |
| `--font-base`| 16       | 20            | 主要內文 ★            |
| `--font-lg`  | 18       | 22            | 卡片標題、子標題      |
| `--font-xl`  | 20       | 26            | 區塊標題             |
| `--font-2xl` | 24       | 28            | 頁面標題             |
| `--font-3xl` | 32       | 36            | Hero                |

行高 token：`--lh-tight` 1.3（短標題）、`--lh-base` 1.6（內文）、`--lh-loose` 1.8（長者）。

## 4. 分級醫療色彩編碼（台灣分級醫療制度）

台灣分級醫療：自我照護 → 基層 → 區域 → 醫學中心 → 急診。
顏色語義從「冷靜、可自理」遞進到「需立刻處理」，**前端不可硬寫 hex**，
透過 `--sev-*` token + `data-severity` attribute 套用。

| 級別          | Token           | 顏色 / 視覺處理                  |
|---------------|----------------|----------------------------------|
| 自我照護      | `--sev-self`    | 綠：穩定、薄荷                    |
| 基層診所      | `--sev-clinic`  | 醫療藍：日常可預期                 |
| 區域醫院      | `--sev-regional`| 深海軍：強調但不警示                |
| 醫學中心      | `--sev-medical` | 紫：複雜、需專業                    |
| 急診          | `--sev-er`      | 紅「點」不紅「面」：邊框 / icon / chip |

## 5. 深淺主題策略

預設**跟隨系統**（`prefers-color-scheme`）。但下列頁面在內容性質上需強制覆寫：

| 情境              | 主題      | 理由                                                          |
|-------------------|-----------|---------------------------------------------------------------|
| 症狀分析 `#symptoms`| 強制 light | 醫療決策內容需高對比，避免深底反白造成視覺疲勞                          |
| 報告 `#reports`     | 強制 light | 同上，PDF 預覽需貼近列印效果                                       |
| 夜間提醒 `/medications`| 強制 dark  | 夜間吃藥提醒，深底減少藍光與刺眼                                     |
| Landing             | 保留現有切換 | Landing 是品牌頁，兩主題都有設計                                    |

## 6. 反模式（明確禁止）

- ❌ **大面積 Glassmorphism**：在繁中文字上 backdrop-filter 會降低可讀性，違反第 6 條長者友善。
- ❌ **Neumorphism**：低對比，違反 WCAG。
- ❌ **純黑 (#000) 底**：在台灣文化中與喪儀關聯，且增加 OLED 邊緣鋸齒。
- ❌ **大紅大金組合**：節慶感，不適合長期醫療場景。
- ❌ **每頁多個 CTA 顏色**：分散注意力，違反 Hick's Law；一頁最多 1 個主 CTA 色。
- ❌ **警示色當底色**：紅色背景的整張卡片會引發焦慮，反而降低使用者回應急性事件的意願（Tun 2025 JMIR）。

## 7. 長者模式（elder-mode）特化

啟用方式：`html.elder-mode` 或 `html[data-mode="senior"]`（與既有設定對齊）。

調整項：

- 暖白底（#FFFBF5）取代純白，HSL L≈97%。降低眩光，對白內障與老花更友善。
- `--accent` 加深 1 階（`#3B7AAA`）以維持對比度。
- `--danger` 略暗一階（`#B84444`），避免飽和紅刺激。
- 字級 +1 階；行高 1.75。
- 所有按鈕 `min-height: 56px`、`min-width: 56px`。
- Icon `stroke-width: 2.5`（接近 filled），辨識度高於 outline。

## 8. 對應的設計憲法

| 憲法                       | 本研究落實項                                          |
|----------------------------|-----------------------------------------------------|
| 1. PWA 原生體驗             | 預設跟隨系統主題；觸控目標符合 mobile 規範             |
| 2. 可信任、可解釋的 AI       | 醫療藍 + 警示用點不用面，AI 建議不會被恐嚇色綁架       |
| 3. 可客製化的提醒           | 夜間提醒強制 dark 模式                                 |
| 4. 跨院整合視圖             | Bento Grid + ICD-10 chip 用 token 化的 severity 色 |
| 5. 醫病共決                 | 選項用 token 顏色區分，不堆疊在一個爆炸畫面            |
| 6. 長者／家屬模式           | elder-mode 完整實作（字、行高、tap、對比、stroke） |
| 7. 本地化與文化敏感         | 避純黑、避大紅大金；繁中字級 +1 階                     |
