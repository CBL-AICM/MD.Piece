# MD.Piece 產品方向：從「醫療工具」到「疾病長期作業系統」

> 本文件記錄 MD.Piece 的下一階段策略定位。
> 與 [`product-principles.md`](product-principles.md) 並列：
>   - `product-principles.md` 答「WHAT 要做什麼才算好」（7 條設計憲法 + 3 個策略場景 A/B/C）
>   - 本文件答「WHERE 我們往哪走」（產品定位 + 5 條改善方向 + 技術紀律）
>
> 任何新 PR 在自我檢核時，除了對照 7 條設計憲法，也應思考是否在推進這 5 條改善方向之一。

---

## 一、產品定位

### 現在像什麼

> 「功能很多的醫療工具」

### 要變成什麼

> **「真正幫患者管理疾病狀態的系統」**
>
> Longitudinal Disease Operating System
> ——把疾病變成可以被長期追蹤、理解、預測的系統。

### 核心判準

不是「功能數量」，而是：

> **「病人現在到底該做什麼。」**
>
> 打開 app 3 秒內知道現在狀況。

成熟產品的重點不是「功能很多」，而是：

> **「病人在最虛弱的時候，還是知道怎麼用。」**

所以下一階段的設計原則：**刪功能、不加功能**。

---

## 二、5 個改善方向（依優先序）

### 方向 1：首頁改成 Today Dashboard

**現況**：首頁偏功能入口。

**目標**：病人「今天的狀態中心」。

打開 app 3 秒內看得到：

- 今日服藥（已打 / 剩餘）
- 下一次用藥時間
- 症狀變化趨勢
- 是否有 flare 風險
- 未完成事項
- 是否需要提前回診

### 方向 2：加入「病情惡化偵測」

**現況**：只有紀錄感、缺少「疾病正在變化」的感覺。

**目標**：系統能判斷病情是否正在惡化。

flare risk score 的訊號來源：

- 疼痛變化
- 睡眠下降
- 呼吸問題
- 感冒後變化
- 心率異常

### 方向 3：患者 UI 要更「安定」

**現況**：偏工具感、效率感。

**前提**：病人通常 **很焦慮、很疲累、認知負荷低**。

**UI 應做到**：

- 大字
- 少資訊
- 單任務
- 柔和動畫
- 降低壓力感

可保留的安定感元素：櫻花、草地、柔色系、呼吸感動畫。

> 醫療 UI 不只是效率，還包含情緒穩定。

### 方向 4：「提前回診」升級成 Smart Escalation System

**現況**：只是一顆按鈕。

**目標**：當系統發現以下任一訊號時，**主動**：

- 疼痛持續增加
- 呼吸變差
- 心跳異常
- 用藥沒改善

**自動產出**：

1. 建議提前回診
2. 整理就診摘要
3. 匯出 PDF
4. 產生 timeline
5. 幫忙整理要跟醫師說的話

**核心價值**：幫病人知道**什麼時候該去醫院**。

### 方向 5：建立「醫病共享摘要」（Doctor Snapshot）

**前提**：醫師不會想看很多資料，他們要的是**「30 秒看懂病況」**。

**Doctor Snapshot 內容**：

- 疼痛趨勢
- 睡眠變化
- 呼吸事件
- 用藥遵從率
- flare score

**搭配 Timeline**（例）：

```
5/20  感冒
5/21  心跳變快
5/22  類固醇增加
5/24  肌力下降
```

**核心目標**：**降低醫病資訊摩擦**。

## 三、技術紀律補強

> 這 3 條是上述 5 個方向都依賴的技術前提。

### 1. Offline-first

住院網路常很差。PWA 應支援：

- 離線使用
- 本地儲存
- 網路恢復後同步

### 2. 減少通知疲勞

不要一直提醒。應做**智慧提醒**：

- 狀況差 → 提高提醒頻率與優先度
- 正常時 → 降低干擾

### 3. Accessibility 預設考慮

患者狀態經常是：**疲勞 / 手抖 / 視力差 / 呼吸不舒服**。

所以：

- 按鈕大
- 字大
- 深色模式可讀性高
- 少小 icon

## 四、PR 自我檢核增補

繼承 `product-principles.md` 第四節的格式之外，**新功能 PR 應額外回答**：

1. 推進 5 個改善方向的哪一條？（如不在 5 個方向內，請說明為何仍值得做）
2. 是否符合「打開 app 3 秒內知道狀況」的核心判準？
3. 這個改動是「加功能」還是「刪功能 / 簡化」？若是加功能，是否有評估過用更少資訊達成同樣價值？
4. UI 是否考慮到病人「焦慮 / 疲累 / 認知負荷低」的狀態？
5. 是否觸及 Smart Escalation / Flare Detection 資料管線？若是，是否能餵進「自動建議提前回診」流程？
