# 產品復發引擎 × 過去合成的 3200 位假患者 — 預測驗證報告

> 把「過去合成過的」3200 位虛擬患者餵進**產品線**復發風險引擎
> [`backend/utils/recurrence.py`](../../backend/utils/recurrence.py)，量化它的預測有沒有對到真實復發。
> 重現腳本：[`tests/run_recurrence_on_synth_cohort.py`](../../tests/run_recurrence_on_synth_cohort.py)

## TL;DR

- **資料**：16 疾病 × 200 人 × 180 天的合成世代（`md_piece` 模擬器，`base_seed=2024`，決定性可重現）。
- **方法**：`as_of` 釘第 165 天 → 引擎只看前 60 天預測「未來 14 天復發」；用模擬器真實的第 166–179 天 `in_flare` 當 ground truth。
- **結果**：產品引擎的風險分數對真實復發有**中等且方向正確**的判別力——
  **AUROC = 0.70**、會復發者平均風險 **44.3 vs 19.2**、風險 band 與真實復發率**單調對應**（7.7% → 16.8% → 37.5%）。
- **誠實註記**：asthma 等隨機觸發型 flare 本質上難由病程前兆預測（oracle 也≈0.5），會拉低整池數字——這是疾病性質，非引擎缺陷。

## 方法

| 環節 | 設定 |
|---|---|
| 世代來源 | `md_piece.cohort_generator.generate_cohort`，16 疾病 × 200 人 × 180 天，`base_seed=2024` |
| 決定性 | seed 固定 → 與「過去合成過的」那批逐一相同（非重新捏造） |
| 橋接 | 每位患者逐日時序（活動度 / `in_flare` / 漏服 / 觸發因子 / 生物標記）→ 引擎吃的 Supabase 表列（情緒 / 用藥 / 症狀 / 睡眠 / 飲食 / 就診 / 檢驗） |
| 活動度→主觀訊號 | **疾病別絕對刻度**（`baseline.activity → flare.threshold`，同病所有人共用一把尺，不做 per-patient 正規化） |
| 預測時點 | `as_of` = 第 165 天（保留 14 天未來當 ground truth） |
| Ground truth | 第 166–179 天是否出現 `in_flare`（任一日復發 / 新發作 onset 兩種標籤） |

橋接是「同一個底層惡化狀態、多訊號同時變差」的忠實展開，每條對應都寫在腳本 `adapt()` 內（透明原則）。

## 結果（3200 人）

### 引擎輸出分布

| 指標 | 數值 |
|---|---|
| 覆蓋 | 可預測 **3200 / 3200**、冷啟動 0、全員綁定疾病文獻錨 |
| 風險 band | low **2436** / medium **297** / high **467** |
| 平均風險 % | 22.4 |
| 趨勢 | 持平 2055 / 上升 659 / 下降 486 |

### 預測 vs 真實復發

| 指標 | 數值 |
|---|---|
| 實際復發基率 | 任一日復發 12.9% ｜ 新發作 8.9% |
| **AUROC（風險% → 未來 14 天復發）** | **0.699**（新發作 0.577） |
| 平均風險%：會復發 vs 不會 | **44.3 vs 19.2** |

### 風險 band × 實際復發率（核心驗證：單調遞增）

| band | n | 實際復發率 |
|---|---:|---:|
| low | 2436 | **7.7%** |
| medium | 297 | **16.8%** |
| high | 467 | **37.5%** |

→ 引擎判為高風險的人，真實復發率是低風險組的近 5 倍。

## 過程中修掉的一個橋接 bug（透明）

第一版把活動度做 **per-patient 正規化**，等於丟掉「絕對嚴重度」——慢性高活動但穩定的人被誤判成「正常」，
引擎的情緒 level / 睡眠 / 檢驗等絕對值因子因此讀不到嚴重度 → **AUROC = 0.498（等同亂猜）**。

先用 oracle 診斷確認「未來 flare 其實可由活動度預測」（RA/SLE oracle AUROC ≈ 0.68–0.75），
再改成**疾病別絕對刻度** → **AUROC 0.498 → 0.70**。這是橋接忠實度的修正，**未動引擎一行**。

## 限制與註記

- band 指定為**疾病分級啟發式**（非逐病文獻復發率）；目的是讓引擎的疾病錨路徑有覆蓋，非臨床定論。
- 隨機觸發型疾病（asthma 等）的 flare 缺乏可觀察的病程前兆，oracle 上限即≈0.5，會稀釋整池 AUROC。
- 進行型疾病（IPF/SSc）在 14 天窗內 flare 極少，屬資料性質，不代表引擎漏抓。
- 這是「產品引擎（決定性、相對自身 + 絕對 level 的啟發式）」的評估，**非** sim 分支那顆 LSTM 模型。
