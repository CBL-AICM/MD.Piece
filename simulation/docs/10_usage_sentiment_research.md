# 使用率・使用心聲・使用結果 整合研究（Usage · Sentiment · Outcome）

延伸主研究（紀錄保真度）至**使用面**：採用率、留存、使用者心聲（PRO/滿意度），並與紀錄結果整合。
資料源自文獻校準版 run（config `713d8a608280`，3,200 人）。

> ⚠ 本層為**衍生分析**，不是決定性引擎輸出。使用者心聲（滿意度/信任/NPS）是依文獻**結構**建模的
> 病患報告結果（PRO），屬專家判斷，標記 `validation_required`；質性「使用心聲」為**模擬示意**，
> 非真實病患發言。真實量測請對接提案既有量表（MAUQ / SUS / CARE / Trust，見 `02_實驗設計`）。

## 1. 方法：心聲模型如何錨定文獻（PubMed）

依兩篇 PubMed 來源的**結構**（非數值）建模：

- **MAUQ**（Mustafa 2021, *JMIR Mhealth Uhealth*, PMID 33538704, [DOI](https://doi.org/10.2196/24457)）：
  mHealth App 可用性 = {易用性 ease-of-use、介面與滿意度 interface/satisfaction、實用性 usefulness}
  三個子構面（Cronbach α=0.946）。→ 滿意度 = f(實用性, 易用性, 溝通效益)。
- **Kruse 2015**（*J Med Internet Res*, PMID 25707035, [DOI](https://doi.org/10.2196/jmir.3703)）：
  病患入口網/PHR 於慢性病的系統性回顧——**最常見正向主題＝醫病溝通（37%）**；最常見負向＝
  安全疑慮與**操作門檻（41%），對不熟科技者尤甚**。→ 正向心聲歸於溝通效益、負向歸於操作門檻／流失。

**操作化**（每位病患，衍生自既有指標）：
- 實用性 = MD.Piece 紀錄品質（臨床重建分數）
- 易用性 = 科技識讀（＋照顧者加成）
- 溝通效益 = 模擬醫師理解度
- 滿意度（MAUQ 式 0–100）= 基線 + 受益程度（ΔCRS）+ 易用性 + 溝通效益 + 雜訊；**未採用者偏低**
- NPS：推薦者 ≥80、中立 60–79、貶損者 <60；續用意願由滿意度推導

| 指標 | 值 |

| 相關係數 corr(受益, 滿意度) | **0.75** |

**依人格平均滿意度**（清楚的交叉）：

| 人格 | 滿意度 | | 人格 | 滿意度 |
|---|---|---|---|---|
| 完美記錄者 | **75** | | 一般 | 50 |
| 照顧者代管 | **71** | | 長者低識讀 | 40 |
| 焦慮型 | 63 | | 低參與 | 36 |
| 症狀驅動 | 49 | | 科技迴避 | **30** |

> 全族群 NPS −48 反映「**無差別大規模部署**」會留下許多未採用／不滿意者；這正是負面但誠實的結果。
> 受益與滿意度高度相關（r=0.75）——**用得越深 → 受益越多 → 越滿意**（`figures/usage_integrated.png`）。

## 4. 使用結果整合（Usage → Sentiment → Outcome）

使用漏斗（全族群）：下載 100% → 啟用 66% → M3 活躍 40% → 滿意(60+) 39% → 推薦者(80+) 13%
（`figures/usage_integrated.png`）。

**核心整合結論**：使用面（採用/留存）、心聲面（滿意/推薦）、結果面（紀錄保真度 ΔCRS）三者
**呈現同一條交叉線**——照顧者中介與高參與者「用得深、受益大、滿意高、會推薦」；科技迴避與
低參與者「不採用、無受益、不滿意、會流失」。這與主研究的紀錄保真度交叉**完全一致**，三個面向
**互相強化**同一個可行動結論：

> **MD.Piece 的價值有條件且可被瞄準。** 應聚焦照顧者中介與高參與族群並強化「易用性／onboarding」
> 以擴大採用；無差別部署會同時拉低保真度、滿意度與留存。

## 5. 限制與真實驗證路徑
- 心聲模型為文獻**結構**錨定的專家判斷，數值未經真實量測（`validation_required`）；質性心聲為模擬示意。
- 真實驗證對接提案既有量表：**MAUQ**（可用性）、**SUS**、**CARE**（醫病關係）、**Trust**、**eHEALS**
  （見 `02_實驗設計`）；建議於實驗一/三蒐集真實滿意度、NPS、留存以替換本層假設值。
