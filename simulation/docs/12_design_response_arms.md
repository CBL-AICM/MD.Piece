# 設計回應 arm — 各設計支柱的邊際效益（roadmap 排序）

把 `docs/11_patient_usage_problems.md` 的 5 大設計支柱轉成模擬中的**參數介入**，跑全管線量化各支柱
對三個結果面向的邊際效益：**紀錄保真度 ΔCRS**、**醫師理解度**、**使用者滿意度／留存**。
（基準＝文獻校準版 config 713d8a608280；每情境 n=900。圖：`figures/design_arms.png`。）

> ⚠ 介入幅度為設計假設（如 onboarding 0.70→0.92、OCR 擷取率 0.40）；OCR 為新增機制（`validation_required`）。
> 數字為模擬上限估計，非臨床主張。

---

## 1. 支柱 → 參數介入對照

| 支柱 | 設計 | 模擬參數介入 |
|---|---|---|
| **A 降門檻** | onboarding 簡化、長者/家屬模式 | `onboarding_completion` 0.70 → 0.92 |
| **B 降負擔** | OCR 被動擷取（文件上傳） | `ocr_capture.enabled` = true（擷取率 0.40，文件型事件、與參與度無關、日期準確）|
| **C 智慧提醒** | 客製化分眾提醒 | `notification_recovery.max_recovered_frac` 0.30 → 0.50 |
| **D 醫病閉環** | Doctor Snapshot、回診前摘要 | 醫師 `review_probability` 0.55→0.75、`reading_budget` 200→320、`noise_penalty` 0.6→0.30 |
| **E 信任與留存** | 隱私優先、跨院整合 → 留存↑ | `retention.median_lifetime_days` 75 → 130 |

## 2. 各支柱的邊際效益（vs 基準）

基準：ΔCRS **+0.108**、受益 64%、滿意度 49、M3 留存 40%、醫師理解度 0.220。

| 支柱 | ΔΔCRS（保真度）| Δ醫師理解度 | Δ滿意度 | Δ M3 留存 | 主要強項 |
|---|---|---|---|---|---|
| **B 降負擔（OCR）** | **+0.060** | +0.057 | +2.0 | +0.0 | **紀錄保真度第一** |
| **C 智慧提醒** | +0.046 | +0.030 | +1.5 | +0.0 | 保真度第二 |
| **A 降門檻** | +0.028 | +0.019 | **+8.3** | **+11.6** | **採用/滿意/留存第一** |
| **E 信任與留存** | +0.016 | +0.008 | +1.0 | +5.9 | 留存 |
| **D 醫病閉環** | +0.000 | **+0.127** | +1.2 | +0.0 | **醫師理解度第一（最後一哩）** |
| **ALL 全部組合** | **+0.130** | **+0.257** | **+15.3** | **+19.6** | 全面 |

## 3. 核心發現：沒有單一支柱通吃——各自最佳化不同結果

- **要更完整準確的紀錄（ΔCRS）** → **B（OCR 被動擷取）** 最有效，其次 C。因 OCR 與參與度無關，
  能補回流失者漏掉的文件型事件（檢驗、出院、處方、輸注）。
- **要醫師真正讀懂（臨床價值的最後一哩）** → **D（Doctor Snapshot）** 壓倒性領先（Δ理解 +0.127，
  其他支柱 < 0.06）。D 幾乎不動 ΔCRS——它改善的是「醫師理解」而非「紀錄本身」。
- **要更多人採用、更滿意、留得住** → **A（降門檻 onboarding）** 第一（滿意 +8.3、留存 +11.6）。
- **全部組合** 使 ΔCRS 由 +0.108 升至 **+0.238**（≈ 先前求得的「全面採用上限 +0.235」），
  醫師理解度 +0.257、滿意度 +15、留存 +20——**完整重設計可逼近理論上限**。

## 4. Roadmap 建議（依目標決定優先序）

| 若主要目標是… | 優先支柱 |
|---|---|
| 臨床決策品質（醫師讀懂）| **D → B → C** |
| 紀錄完整準確度 | **B → C → A** |
| 採用率與留存（規模化）| **A → E → C** |
| 全面（資源足夠）| **ALL**（逼近上限，邊際效益彼此互補、近似可加）|

> 這**精煉**了先前單純依 Sobol 的排序（C 通知補登為翻轉關鍵）：把「結果面向」分開看後，
> **D 對臨床理解、B 對紀錄、A 對採用** 各為其面向的最佳投資。建議產品 roadmap：
> **先 D（snapshot，臨床價值）＋ A（onboarding，擴大採用）為雙主線，B/C 為紀錄強化，E 為留存底盤。**

## 5. 與真實驗證的對接
- 模擬已可比較各支柱（本文）；真實驗證對接提案 `02_實驗設計` 的三項實驗與量表（MAUQ/SUS/CARE/Trust/eHEALS）。
- 建議下一步真實實驗以 **D（snapshot）與 A（onboarding）** 為優先介入臂，因其在模擬中對「臨床理解」與「採用」邊際效益最大。

## 限制
介入幅度為設計假設；OCR 機制為新增、需真實擷取率校準；各支柱以獨立介入估計，組合效益假設近似可加
（ALL 結果支持此假設）；數字為模擬上限。重現：`python -m simulation.run_design_arms`。
